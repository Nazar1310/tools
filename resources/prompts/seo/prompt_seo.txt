TASK:
Generate production-ready SEO HTML content for a tool page based on the provided TOOL DATA.

GOAL:
Create high-quality SEO content that improves search visibility, satisfies user intent, supports tool usage, and provides real informational value.

INPUT DATA:
You will receive TOOL DATA containing:

- name
- title
- description
- SEO fields (context only)

IMPORTANT:

- The page already contains an H1
- DO NOT generate any <h1>
- DO NOT repeat the page title as a heading
- Focus ONLY on SEO content
- Do NOT generate UI components or interactive functionality
- Output must be production-ready HTML

CONTENT STRATEGY REQUIREMENTS:

Before generating content:

1. Identify likely search intent
2. Determine tool category
3. Match content structure to actual user needs
4. Avoid generic templated SEO content

The result should feel tailored to the tool, not mechanically generated.

HTML STRUCTURE REQUIREMENTS:

Wrap all primary SEO content inside:

<section class="seo-section">

Use semantic HTML only:

- h2
- h3
- p
- ul
- ol
- strong
- em
- table
- tbody
- tr
- td
- thead
- th

If FAQ is included, wrap it separately:

<section class="faq-section">

SECTION REQUIREMENTS:

Choose only sections that provide real value.

Possible sections:

1. Introduction
Purpose:
- explain what the tool does
- clarify user benefit
- satisfy quick informational intent

2. How It Works
Include when useful:
- formulas
- logic
- transformations
- methodology
- assumptions

3. How to Use
Include when useful:
- practical usage steps
- concise guidance
- only if not duplicating obvious UI

4. Example
Include when useful:
- realistic practical example
- result interpretation

5. Understanding Results
Include when useful:
- explain outputs
- interpretation guidance
- confidence/limitations

6. Common Mistakes
Include when useful:
- input errors
- interpretation mistakes
- assumption misunderstandings

7. Limitations / Constraints
Include when useful:
- precision limitations
- unsupported cases
- methodological boundaries

8. Practical Use Cases
Include when useful:
- realistic applications
- problem-solving context

FAQ RULES:

FAQ is NOT mandatory.

Include FAQ ONLY when it provides genuine value for:

- search intent coverage
- user objections
- interpretation questions
- common search queries
- edge-case clarification

FAQ structure:

<section class="faq-section">
  <h2>FAQ</h2>
  <h3>Question</h3>
  <p>Answer</p>
</section>

WRITING RULES:

Use:

- concise paragraphs
- clear structure
- direct explanations
- natural language
- semantic relevance
- scan-friendly formatting

Avoid:

- keyword stuffing
- generic filler
- repetitive statements
- exaggerated claims
- robotic AI phrasing

STRICTLY AVOID phrases like:

- easy-to-use
- powerful tool
- quickly and easily
- designed to help
- simple solution
- seamless experience
- user-friendly platform
- save time and effort

SEO EXPECTATIONS:

Naturally support:

- primary search intent
- semantic keyword relevance
- related query coverage
- long-tail discoverability
- featured snippet potential
- People Also Ask alignment

QUALITY BAR:

The content must feel like it was created by an experienced SEO strategist for a real production website.

It must NOT feel like generic AI-generated filler.

OUTPUT:

Return ONLY valid HTML.
No explanations.
